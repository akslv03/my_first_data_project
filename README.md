# 1. Бизнес-анализ (Business Understanding)

**Организационная структура:**
* **Заказчик сервиса:** Медицинская клиника.
* **Основной пользователь:** Врач-рентгенолог, диагност, онколог.

**Бизнес-цель проекта:**
Повышение точности и скорости постановки диагнозов при глиомах низкой степени злокачественности. Сервис должен стать системой помощи в принятии врачебных решений. Он автоматизирует рутинную работу (обводку контуров опухоли) и предоставляет дополнительный контекст в виде базы исторических похожих случаев.

**Существующие решения:**
* *Текущее решение:* Ручная разметка МРТ-снимков врачом.
* *Чем не устраивает:* Занимает много времени, подвержено человеческому фактору. Врачу трудно держать в голове тысячи прошлых снимков пациентов, чтобы сравнивать их с текущим случаем при назначении лечения.

## 1.1 Текущая ситуация (Assessing current solution)

**Ресурсы и данные:**
* **Железо:** Для ресурсоемкого этапа обучения модели будут использоваться бесплатные облачные мощности (например, Google Colab с GPU).
* **Хранение данных:** В качестве источника используется открытый датасет с платформы Kaggle: "Brain MRI segmentation". Данные будут скачаны локально для тестов, доступ к ним свободный, докупать внешние данные не требуется.

**Возможные риски проекта и план действий по их уменьшению:**
1. **Риск: Не уложиться в сроки.**
   * *План действий:* Не тратить время на изобретение новой архитектуры сети, а использовать архитектуры, которые хорошо зарекомендовали себя в данном домене.
2. **Риск: Нехватка вычислительных мощностей локального ПК.**
   * *План действий:* Выбор 2D-подхода вместо 3D-сегментации для снижения вычислительной нагрузки.

## 1.2 Решаемые задачи с точки зрения аналитики (Data Mining goals)

**Метрики для оценки результата моделирования:**
Для задачи медицинской сегментации классические метрики вроде Accuracy не подходят из-за сильного дисбаланса классов (фон занимает больше места, чем опухоль). Будет использоваться:
1. **Dice Similarity Coefficient (DSC)** — для оценки качества выделения маски опухоли нейросетью.

**Критерии успешности модели:**
* **Минимальный порог:** Коэффициент Dice > 0.75 на тестовой выборке.
* **Оптимальный результат:** Коэффициент Dice > 0.85 на тестовой выборке, а сервис корректно возвращает маску опухоли и список наиболее похожих случаев.

## 1.3 План проекта (Project Plan)

Проект будет выполняться по методологии CRISP-DM:
1. **Business Understanding (Текущий этап):** Определение цели, выбор темы, постановка задачи, анализ рисков.
2. **Data Understanding (Анализ данных):** Скачивание датасета с Kaggle, базовый разведочный анализ данных (EDA), изучение распределения масок и размеров изображений.
3. **Data Preparation (Подготовка данных):** Написание пайплайна загрузки данных, аугментация снимков, нормализация.
4. **Modeling (Моделирование):** Адаптация готового baseline (U-Net) под свои нужды, обучение модели на GPU в облаке, реализация извлечения эмбеддингов для рекомендательной системы.
5. **Evaluation (Оценка результата):** Расчет метрики Dice на отложенной тестовой выборке, визуальная оценка предсказаний и релевантности найденных "похожих" снимков.
6. **Deployment (Внедрение):** Создание микросервисной архитектуры (FastAPI + векторная БД), оборачивание в Docker, настройка CI/CD-пайплайна в GitLab для проверки качества кода.
